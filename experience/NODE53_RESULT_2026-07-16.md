# Node 53 - DM Routing, Context Cost And Outbound Safety

Status: complete and deployed on 2026-07-16.

## Incident Closed

- The 2026-07-15 News-group send was traced to outbox 139.
- A DM payload containing embedded historical XML was misread as a cross-channel send command.
- The old implementation selected the global latest group draft and used `force=True`, bypassing stale and internal-content guards.
- Blank `dm:` conversations also allowed provider/context errors to reach outbox and, historically, real delivery.

## Core Changes

- Blank DM input is normalized to `dm:<sender wxid>` when the sender is known.
- Unrecoverable blank DM input is rejected before message insertion.
- Blank conversation targets are rejected before outbox creation.
- Historical blank DM rows are blocked before Hermes response generation.
- The implicit "send latest group draft" path is disabled.
- Internal provider/context error guards cannot be bypassed with `force=True`.
- Added guards for context-length, compression, token-limit and HTTP 400 failures.
