# Node 21 - Bounded Public Web Evidence For Group Replies

Status: complete and deployed on 2026-07-13.

## Mainline Gap Closed

Group policy allowed public-web material, but the reply pack had no bounded, attributed public evidence. Group replies can now read relevant cached public evidence with source and time attribution.

## Existing Component Extended

- Extended `UnifiedMemoryStore.group_reply_context()` and the existing `raw_web_evidence` store.
- Reused the existing group policy, Hermes reply pack, Core instances and middleware route.
- No second search engine, reply engine, evidence table or worker was created.

## Boundaries And Fallback

- Query is derived only from the current group request and limited to six terms.
- Maximum five records at the Store boundary; group replies request three.
- Each excerpt is limited to 900 characters.
- Only successful cached public evidence with a source URL is returned.
- Groups without `public_web` in `allowed_sources` receive no public evidence.
- Evidence is reference material and cannot override exact-group confirmed records.
- Empty/no-match/failure fallback is an empty evidence list; no invented fact.
- This node performs no live network fetch inside the synchronous reply path.
