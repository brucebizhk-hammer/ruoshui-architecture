# Node 25 - Asynchronous Public Evidence Refresh Request Queue

Status: complete and deployed on 2026-07-13.

## Mainline Gap Closed

Group replies could identify missing or stale public evidence but had no durable, deduplicated way to request later refresh. The existing group reply context now stages a refresh request without performing network work in the synchronous reply path.

## Existing Component Extended

- Extended `group_reply_context()` and Unified SQLite.
- Reuses the existing Provider/Worker evidence pipeline for the later claim stage.
- No second reply engine or evidence store.

## Delivered

- `public_evidence_refresh_requests` with request key, group/account scope, normalized query, reason, priority, status, attempts and audit timestamps.
- Partial unique index deduplicates pending/claimed requests.
- Missing evidence stages reason `missing`.
- All-stale/all-unknown evidence stages reason `stale_or_unknown`.
- Fresh/recent evidence and groups without `public_web` permission do not stage requests.
- Current reply continues with conservative missing/stale wording; no network wait.
