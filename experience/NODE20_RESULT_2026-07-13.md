# Node 20 Result - Isolated Group Mention Reply Context

Status: complete

## Mainline Gap Closed

The existing `tool_agent` reply pack used generic local knowledge and could inherit legacy contact memory. Group mentions now receive an explicitly isolated context assembled inside the existing reply pipeline.

## Existing Component Extended

- Extended `build_memory_pack()`, `auto_send_scope()` and existing group profile/policy APIs.
- Reused the same Hermes session, outbox, outbound guard and auto-send path.
- Did not create a second reply engine.

## Delivered

- Group reply context contains only the exact group's confirmed business records.
- Current-group recent history remains available; other group conversations are excluded by conversation ID.
- Public/industry knowledge sources remain available; local `business_facts` are excluded for group role.
- Personal profile, private notes and contact memories are absent from the group pack.
- Candidate/rejected extractions are absent; only confirmed/corrected records are included.
- Group-role Hermes memory writes are disabled so the extraction/review pipeline remains the sole owner of group facts.
- Auto-send now requires runtime allowlist plus an existing group profile with `reply_enabled=true`.
