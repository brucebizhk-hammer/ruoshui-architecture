# Node 24 - Account-Origin Authorization Guard

Status: complete and deployed on 2026-07-13.

## Mainline Gap Closed

Legacy Songsheng-account relationships were labelled as Ruoshui customer profiles and active authorizations. They could therefore be mistaken for current Ruoshui friends. Runtime authorization now requires explicit account-origin verification.

## Existing Component Extended

- Extended existing `customer_profiles` and `customer_authorizations` additively.
- Reused the existing authorization lookup and upsert API.
- No duplicate customer, identity or authorization system.

## Delivered

- Profile fields: `relationship_origin`, `origin_account_id`, `origin_verified`.
- Authorization fields: `relationship_origin`, `origin_verified`, `origin_verified_at`.
- Legacy rows default to `legacy_unverified` and remain readable/auditable.
- Runtime authorization accepts only `origin_verified=1`.
- Authorization API can explicitly create a verified `ruoshui_current_friend` record.
