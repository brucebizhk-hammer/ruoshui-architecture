三、工具体系与职责边界

大类工具/组件当前作用状态不能做什么
公开发现Exa Search、RSS、Direct Search发现公开页面与候选入口运行不能直接写客户、不能发微信
网页获取Jina Reader、Scrapy、Playwright、requests读取静态/动态页面、附件和正文部分运行不绕过登录、付费墙或平台限制
内容清洗Trafilatura、BeautifulSoup、OCR正文、表格、图片文字抽取运行抽取结果不是客户事实
身份解析phonenumbers、RapidFuzz、本地规则电话标准化、模糊匹配、去重建议运行模糊结果不得强行合并身份
微信通道WeChat 4.1.8 Hook DLL + Middleware HTTP群列表、成员、联系人详情、消息、添加请求添加暂停不得绕过严格 v3/v4 和账号校验
业务控制Ruoshui Core评分、授权、策略、审计、Action Gateway主干运行外部 Provider 不能替代 Core 裁决
数据底座SQLite / FTS5唯一客户、证据、队列、关系和审计事实库权威报表和 HTML 不是数据源
调度监控APScheduler、任务计划、Prometheus周期任务、状态、熔断、运行指标部分运行不能把计划参数误报为正在运行
对话与人设Hermes、本地规则、内容运营计划破冰、意向识别、朋友圈素材草稿待添加后启用不处理开发进度同步
开发通道CodexBridge / ClawBot 官方 iLink开发任务、节点和进度归口运行不进入若水客户记忆或业务微信

四、端到端主流程

流程以“候选进入待添加库”为本图终点；添加后的自动聊天和客户经营属于下一阶段。
1. 信源发现网页、群成员、人工线索、数据库、社交与行业平台输出：URL、群成员 ID、手机号、公司名或文章
→
2. 原始证据层保留来源、时间、标题、正文、群与账号raw_web_evidence / group_memberships
→
3. 清洗与身份层正文抽取、电话标准化、公司/人名解析、去重不确定匹配保留候选关系
→
4. Lead 评分层相关度、可联系性、来源质量、风险、时效当前已评分 371 条
→
5. 待添加库按来源、账号、优先级和风险排队历史 198；有效 170；严格就绪 0
6. 严格凭据解析手机号：search → get_contact群成员：get_group_member_contact
→
7. 安全门禁目标匹配 + v3 + v4 + 若水账号 + 单次熔断当前卡在 v3 缺失
→
8. 添加请求add_friend 单人提交、返回检查、客户端存活检查当前暂停
→
9. 结果回写通过、拒绝、无效、待处理历史提交 5，通过 3
→
10. 添加后经营轻量破冰、关键词意向、画像与朋友圈互动不属于当前待添加阶段

五、行业专业群拓客闭环

这是新增主路线：先进入高相关行业群并建立专业可信度，再把群成员转为一对一关系。群内运营和私聊经营使用同一人设，但状态、频率和结果分别记录。
1. 搜索专业群线索行业网站、公众号文章、协会、展会、微信群推荐、短视频评论区输出：群名称、行业、入口、推荐人、有效期
→
2. 群相关性筛选硫磺、化工、贸易、港口、物流、采购、供应链过滤广告群、投资群、无关综合群和高风险入口
→
3. 入群申请队列二维码、邀请链接、推荐人、申请话术、申请状态验证码、付费或管理员确认保留人工处理
→
4. 入群结果回写已申请、已通过、拒绝、二维码失效、待管理员确认通过后建立 group_profile
→
5. 72小时观察默认先沉默观察群规则、活跃时间和业务主题禁止入群后立即批量加人或连续发广告
6. 专业人设建立优先有价值的行情、政策、港口、供需和案例内容群允许时再发；不重复、不硬广、不索取隐私
→
7. 群成员与发言扫描同步成员 wxid、昵称、群身份、公司、发言关键词有货、要货、价格、港口、付款、数量、采购、供应
→
8. 成员相关度评分行业身份 + 群内发言 + 公司 + 角色 + 可联系性群内不发言者只按弱证据处理
→
9. 逐个进入待添加记录来源群、推荐证据、优先级和验证话术同群分散添加，首错即停，结果自动回写
→
10. 私聊与客户分级通过后轻量破冰 → 意向关键词 → C/B/A/S/D客户画像、跟进和朋友圈互动沿用现有规则

六、数据结构：从证据到关系

raw_web_evidence公开页面、来源、正文、状态、错误
group_source_leads（新增）群线索、入口、行业、推荐人、有效期
group_join_requests（新增）申请话术、时间、状态和结果
group_profiles / memberships群、账号、成员身份与来源
group_observation_state（新增）72小时观察、群规、活跃时间、内容状态
lead_pre_pool候选人/公司、联系方式、评分、风险
lead_add_queue添加账号、目标、来源群、优先级、结果
contacts统一自然人/主体规范身份
account_relationship_profiles每个微信账号独立关系
customer_profiles审核后的画像和客户等级
conversations / messages私聊、群聊、来源账号和上下文
action_requests / outbox外部动作与待发送控制
group_content_events（新增）群内容草稿、发布、主题和反馈
audit / result records入群、解析、添加、发送和授权审计
边界：群成员不等于好友，Lead 不等于客户，联系人数量不等于可经营客户；若水和宋生的私聊记忆、好友关系、授权与人设必须隔离。
